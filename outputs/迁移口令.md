# 沈家本研究院项目 — 新电脑迁移口令

## 第一步：网站项目（沈家本研究院官网）

### 1. 把下面这个文件复制到新电脑的网站项目目录下：
`outputs/automations_website.json`

### 2. 在新电脑 WorkBuddy 中打开网站项目，粘贴这段话：

```
请根据 automations_website.json（如文件在 outputs/ 下则读 outputs/automations_website.json）帮我创建里面的自动化任务。
创建前先做两件事：
1. 用 Bash 执行 echo %CD%（Windows）或 pwd（Mac/Linux）获取当前项目路径，替换提示词中所有 {{PROJECT_PATH}} 为这个路径
2. 用 Bash 执行 where python 或 which python3 获取 Python 路径，替换提示词中所有 {{PYTHON_PATH}} 为这个路径（如有 WorkBuddy 管理的 Python 则用那个路径）
然后用 automation_update 工具 mode=create 逐个创建，每个任务的 cwd 设为当前工作目录（不要写死路径）。创建完成后报告全部任务 ID 和名称。
```

## 第二步：公众号项目（文章自动写作发布）

### 1. 把下面这个文件复制到新电脑的公众号项目目录下：
`outputs/automations_wechat.json`

### 2. 在新电脑 WorkBuddy 中打开公众号项目，粘贴这段话：

```
请根据 automations_wechat.json（如文件在 outputs/ 下则读 outputs/automations_wechat.json）帮我创建里面的全部7个自动化任务。
创建前先做两件事：
1. 用 Bash 执行 echo %CD%（Windows）或 pwd（Mac/Linux）获取当前项目路径，替换提示词中所有 {{PROJECT_PATH}} 为这个路径
2. 用 Bash 执行 where python 或 which python3 获取 Python 路径，替换提示词中所有 {{PYTHON_PATH}} 为这个路径（如有 WorkBuddy 管理的 Python 则用那个路径）
然后用 automation_update 工具 mode=create 逐个创建，每个任务的 cwd 设为当前工作目录（不要写死路径）。创建完成后报告全部任务 ID 和名称，确认状态全部为 ACTIVE。
```

---

## 第三步：验证

两步都完成后，在 WorkBuddy 中分别打开两个项目，输入：
```
列出我的所有自动化任务
```
确认共 8 个任务全部存在且状态为 ACTIVE。
