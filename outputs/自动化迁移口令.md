# 新电脑自动化迁移口令

## 使用说明

在新电脑的 WorkBuddy 中，**先打开网站项目文件夹**，然后复制下面第一段话；**再打开公众号项目文件夹**，复制第二段话。

---

## 第一步：网站项目（沈家本研究院官网）

> 在 WorkBuddy 中打开网站项目目录后，复制粘贴这段话：

```
请根据以下配置创建1个自动化任务（活动更新），使用 automation_update 工具，mode=create：

名称：沈家本历史文化园活动更新（公众号抓取）
计划：每周一 09:00 执行（recurring，rrule: FREQ=WEEKLY;BYDAY=MO;BYHOUR=9）
状态：ACTIVE
工作目录：当前工作目录（用 pwd 或 cwd 自动获取，不要写死路径）

执行内容：
使用 wechat-article-search 技能搜索微信公众号文章，按优先级执行：

优先：晓法科普公众号文章（园区官方号，运营方：枕碧文化）
由于该号未入搜狗索引，从以下方向间接获取：
1. 搜索 "枕碧文化" + "沈家本历史文化园" （最近14天）
2. 搜索 "晓法科普" + "沈家本" （最近14天）

其次：园区活动关键词搜索
3. "沈家本历史文化园 党建" (最近14天)
4. "沈家本历史文化园 主题党日" (最近14天)
5. "沈家本历史文化园 联合党建" (最近14天)

从搜索结果中提取在沈家本历史文化园内举办的活动信息。筛选规则：
- 必须在文章中出现「沈家本历史文化园」+ 活动地点明确在该园内
- 活动标题、日期、参与单位、简要描述

与 src/backend-js/seed.js 中现有 events 数据对比去重（标题关键词匹配）。

发现新活动后添加到 seed.js events 数组末尾，活动分类判断规则：
- 含有「党建」「主题党日」「党员」「党支部」「党总支」「党史」「廉政」「纪检」「红色研学」→ category: '党建活动'
- 含有「研讨会」「学术」「年会」「论坛」→ category: '学术会议'
- 含有「研学」「参观学习」→ category: '研学'
- 含有「普法」「宪法」「法治宣传」「文化活动」→ category: '文化活动'
- 含有「签约」「合作」→ category: '签约仪式'
- 含有「开园」「揭牌」「仪式」→ category: '仪式'
- 含「沙龙」「私董」→ category: '沙龙'
- 含「培训」「研习营」「授课」→ category: '培训'

location 统一为「湖州沈家本历史文化园」。
同步更新 dist/events.html（如需新分类），提交 git commit 并 push，部署 CloudStudio。
如无新活动则报告「本周无新活动」。
```

---

## 第二步：公众号项目（每日/每周文章推送）

> 在 WorkBuddy 中打开公众号项目目录后，复制粘贴这段话：

```
请根据以下配置创建7个自动化任务（周一至周日文章推送），使用 automation_update 工具，mode=create，每个任务单独创建：

【任务1】周一｜家本通俗小故事
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=MO;BYHOUR=9;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）

【任务2】周二｜古今普法小专栏
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=TU;BYHOUR=9;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）

【任务3】周三｜角落随笔·法律热点时事
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=WE;BYHOUR=9;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）

【任务4】周四｜商事调解专题研究
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=TH;BYHOUR=9;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）

【任务5】周五｜经典法庭故事
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=FR;BYHOUR=9;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）

【任务6】周六｜刑法人文故事
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=SA;BYHOUR=9;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）

【任务7】每日｜沈家本法治箴言卡片推送
- scheduleType: recurring, rrule: FREQ=WEEKLY;BYDAY=SU;BYHOUR=12;BYMINUTE=0
- status: ACTIVE, cwd: 当前工作目录（自动获取）
- prompt: 执行每日沈家本法治箴言卡片自动推送。运行命令：用当前Python路径执行 wechat_daily_card.py 脚本，完成后报告今日卡片名称与轮换进度。

每个任务的 prompt 内容请参考原项目中的自动化配置（在 .workbuddy/automations/ 目录下可查看），保持原有的写作风格、知识库调用、图片生成、发布流程不变。

创建完成后报告：7个任务的ID和名称，确认全部 ACTIVE。
```

---

## 验证

创建完成后，在 WorkBuddy 中输入：
```
列出我所有的自动化任务
```
确认8个任务（1个网站 + 7个公众号）全部存在且状态为 ACTIVE。
